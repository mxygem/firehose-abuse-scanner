## Commands

```bash
# Run the scanner (uses config.dev.json by default)
go run ./cmd/scanner

# Override config via env vars
BACKPRESSURE_MODE=drop WORKER_COUNT=100 EVENTS_PER_SECOND=10000 go run ./cmd/scanner

# Run all unit tests
go test ./...

# Run a single test
go test ./internal/pipeline/... -run TestName

# Run integration tests (requires Scylla running)
docker compose up -d db
go test -tags=integration ./internal/storage/scylla/...

# Regenerate mocks (requires mockery)
go generate ./...

# Start Scylla
docker compose up -d db
docker compose ps   # wait for STATUS=healthy

# CQL shell
docker exec -it firehose-scylla cqlsh

# Tear down (keeps data volume)
docker compose down

# Wipe data volume
docker compose down -v
```

## Architecture

The scanner is a fan-in worker pool. Events flow: **Firehose â†’ Pipeline â†’ Handler**.

**Firehose** (`internal/firehose`): The `Client` interface has one method, `Subscribe(ctx) (<-chan FirehoseEvent, error)`. Currently only `Simulator` is implemented; a real WebSocket client would satisfy the same interface.

**Pipeline** (`internal/pipeline`): `Pipeline.Run` fans a single source channel into a fixed-size worker pool via a buffered `work` channel. Backpressure is handled at `enqueue`: `drop` mode silently discards when the queue is full; `block` mode stalls the ingestion goroutine. Stats are tracked with atomic counters and logged every 5 seconds.

**Handler** (`pipeline.Handler` interface): `Handle(ctx, FirehoseEvent) error` is called by each worker. The only current implementation is `LogHandler`, which logs the event. The next one to add is `ScyllaHandler` (TASKS.md Â§1.4).

**Storage** (`internal/storage`): The `Storer` interface (`InsertEvent`, `InsertFlagged`, `Close`) decouples the pipeline from the backend. The concrete implementation lives in `internal/storage/scylla`. The `Storer` mock in `internal/storage/mocks/` is generated by mockery and is the correct way to test anything that depends on storage.

**Config** (`internal/config`): Loaded via koanf from `config.json` then `config.<env>.json` (defaulting to `dev`), then overridden by env vars. `ENV` env var selects the environment. Config fields `PostgresDSN` and `RedisAddr` are stale and will be removed in TASKS.md Â§5.2.

## Scylla schema

Three tables, each query-driven:
- `events_by_did` â€” partition `did`, cluster `(received_at DESC, id)` â€” per-author lookups
- `events_by_minute` â€” partition `(kind, bucket_minute)`, cluster `(received_at DESC, id)` â€” time-window scans
- `flagged_events` â€” partition `bucket_hour`, cluster `(received_at DESC, id)` â€” detector hits

Schema is embedded in the binary (`//go:embed schema.cql`) and applied idempotently on every `scylla.New` call.

## Testing conventions

- Unit tests use the `mocks.MockStorer` (testify/mockery); integration tests use `//go:build integration` and require a live Scylla node.
- Integration tests create an ephemeral keyspace (`test_ks_<nanos>`) and drop it in `t.Cleanup`.
- `SCYLLA_TEST_HOSTS` overrides the default `127.0.0.1:9042` for integration tests.
